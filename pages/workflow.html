Agent Workflow

从 PDF 到可复核报告

系统围绕招股说明书这一明确场景，设计“解析、抽取、生成、推送、复核”的智能体工作流。

01

上传招股说明书

用户上传一份公开招股说明书 PDF，并可选填写公司名称。

02

PDF 文本解析

使用 PyMuPDF 提取文本，保留页码，识别扫描版或空文本异常。

03

文本清洗与筛选

去除重复页眉页脚，优先保留公司概况、业务、财务、风险和募资相关内容。

04

LLM 信息抽取

调用 OpenAI-compatible API，要求模型输出严格 JSON 并附证据片段。

05

结构化输出

保存 JSON、CSV 和 Markdown 报告，便于展示、复盘和人工核验。

06

飞书机器人推送

将短摘要推送到飞书群，用于团队协同和讨论提醒。

07

人工复核

对关键数据、证据片段、风险因素和尽调问题进行人工检查。
