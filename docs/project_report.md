# ProspectusInsight Agent AI 智能体课程报告初稿

## 1. 服务对象

本项目服务对象为金融专业学生、行研实习生、投行/咨询/数据分析初学者。他们需要阅读公开披露文件，学习公司分析方法，但通常缺少成熟的信息整理框架和自动化工具。

## 2. 具体问题

招股说明书篇幅长、信息密度高、专业术语多。公司概况、主营业务、核心财务信息、风险因素和募集资金用途分散在不同章节，人工阅读和整理成本较高。初学者容易遗漏重要信息，也难以快速形成结构化公司画像。

## 3. 输入内容

- 招股说明书 PDF。
- 可选公司名称。如果用户留空，则由 AI 从招股说明书文本中识别。

## 4. 输出结果

- 公司画像。
- 业务模式。
- 主要产品或服务。
- 收入来源。
- 核心财务摘要。
- 风险因素。
- 募资用途。
- 尽调问题。
- Markdown 报告。
- CSV/JSON 结构化结果。
- 飞书推送摘要。

## 5. 处理步骤

处理流程为：上传、解析、抽取、生成、保存、推送、复核。

1. 用户上传招股说明书 PDF。
2. 系统使用 PyMuPDF 提取文本并保留页码信息。
3. 对文本进行清洗，去除重复页眉页脚和多余空格。
4. 对长文本进行重点内容筛选，优先保留公司概况、业务、财务、风险、募资用途相关内容。
5. 调用 OpenAI-compatible API，按 Prompt 抽取严格 JSON。
6. 对缺失字段进行容错补齐。
7. 生成 Markdown、CSV、JSON 和飞书摘要文件。
8. 如配置飞书 Webhook，则推送到飞书群。
9. 用户对关键数据、证据片段和结论进行人工复核。

## 6. 模型任务

模型负责理解、分类、抽取、总结和生成尽调问题。具体包括识别公司名称、行业、业务模式、产品服务、收入来源、财务摘要、风险因素、募资用途，并根据资料生成后续尽调问题。

## 7. 工具任务

工具任务包括 PDF 解析、文本保存、CSV/JSON 输出、Markdown 报告生成、CLI 运行、飞书 Webhook 推送、前端静态展示和异常提示。

## 8. 异常情况

- 空文件：提示用户上传有效 PDF。
- 扫描版 PDF：提示需要先 OCR。
- API Key 缺失：提示配置 `.env`。
- Webhook 缺失：跳过推送，不报错。
- 模型输出缺字段：自动填充“未识别”。
- JSON 格式异常：提示模型返回格式异常。
- 信息无法识别：在结果和报告中标注“未识别”，并提示人工复核。

## 9. 测试与迭代

项目通过 CLI 脚本和人工页面测试验证稳定性。测试覆盖正常 PDF、空文件、扫描版 PDF、缺少 API Key、飞书 Webhook 未配置、LLM 输出字段缺失和 JSON 格式异常等场景。迭代重点包括增强错误提示、补齐 JSON 字段、缩短飞书摘要、强调人工复核。

## 10. 总结反思

本项目体现了 AI Agent 在公开资料整理中的价值：它能降低首次阅读门槛，帮助学习者快速形成结构化理解。但招股说明书分析仍然高度依赖原文证据、专业判断和人工复核。项目不做投资结论，而是强调资料整理、证据追踪和审慎表达。
